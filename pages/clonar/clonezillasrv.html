Clonezilla Server

En esta práctica se iniciará un revidor de imágenes Clonezilla desde el que se restaurarán varios equipos de forma simultánea.

Para desarrollar la práctica vamos a necesitar:

  • Un pendrive usb con DRBL.
  • Un equipo ejecutando clonezilla server que contenga las imágenes de disco.
  • Uno o varios portátilos iguales que recibirán la imagen.

Primeros pasos

Crear USB de arranque

Antes de empezar a utilizar Clonezilla server tendremos que crear un medio arrancable desde el que ejecutar la aplicación. Utilizaremos un pendrive de 64GB como medio de arranque.

A continución se detallan los pasos a seguir para crearlo:

#01. El primer paso es descargarse la versión actual de DRBL, la cual podemos obtener en el siguiente enlace:

Web oficial de DRBL

Para la práctica descargaremos la versión stable 2.5.1-16 estable.

#02. También necesitaremos el programa Rufus o similar para crear el USB arrancable. Lo podemos descargar desde su web oficial:

Web oficial de Rufus

Descargaremos la versión portable.

#03. Intoduciomos un pendrive vacío en el ordenador y lo seleccionamos en la interfaz de Rufus.

Selecionamos la imagen ISO de DRBL y escribimos un nombre de etiqueta para identificar el pendrive.

Pulsamos en el botón «empezar»

#04. La imagen de Clonezilla que hemos descargado puede ser escrita de dos modos distintos. Rufus nos recomienda utilizar el modo Imagen ISO, por lo que seleccionamos la opción «Escribir en modo Imagen ISO (Recomendado)» y le damos a «OK».

#05. Ahora nos pregunta si le damos permiso al programa para conectarse a internet y descargar unos ficheros adicionales que necesita para completar el proceso. Pulsamos en «Sí».

Por último, nos recordará que todos los datos contenidos en el pendrive serán borrados. Si estamos seguros de que no hay guardado nada importante, le damos a «Aceptar».

#06. Una vez finalizado el proceso, le damos al botón «CERRAR» y ya tendremos listo un pendrive USB arrancable con DRBL.

Podemos pasar a la siguiente sección «Configuración inicial».

Configuración inicial

Esta parte consiste principalmente en arrancar el ordenador desde el pendrive de DRBL y establecer las preferencias básicas como pueden ser el idioma y distribución de teclado.

#01. Arrancamos el ordenador que hará de servidor desde la memoria USB. Aparecerá la pantalla de bienvenida de DRBL.

De las opciones que nos da useremos la segunda, ya que permite desconectar el pendrive y utilizarlo en otros equipos.

#02. Tras cargar el programa en la memoria RAM aparecerá el menú de selección de idioma. Elegimos el idioma INGLES, ya que el ESPAÑOL nos dará problemas a la hora de seleccionar la ruta de las imágenes.

#03. El siguiente paso es seleccionar la distribución de teclado. El sistema para cambiar la distrubución es bastante confuso, por lo que es mejor dejar la distribución por defecto, teniendo en cuenta que algunas teclas pueden cambiar de posición.

#04. Seleccionamos 0 para arrancar DRBL en modo gráfico.

#05. Abrimos el programa Clonezilla Server.

Contiuamos en la siguiente sección para configurar el servidor.

Restauración desde servidor

Arrancar el servidor

En esta parte iniciaremos un servidor de imágenes desde el cual poder restraurar varios equipos de forma simultánea.

Para llegar hasta aquí, es necesario haber completado las instrucciones de la sección «Primeros pasos».

#01. Al arrancar clonezilla server por primera vez nos dirá que no está configurado y necesita que le demos información.

Pulsamos ENTER para continuar.

#02. En este paso nos pregunta si queremos utilizar el servicio DHCP de la red o uno propio. Escribimos N y le damos a ENTER.

#03. Ahora nos pregunta si queremos que el servicio DHCP asigne direcciones IP sólo a los clientes. Le decioms que sí escribiendo Y y pulsando ENTER.

#04. Seleccionamos dónde están guardadas las imágenes. Como las tenemos en un dico externo USB, selecionamos LOCAL_DEV.

#05. Pulsamos Enter y aparecerá una lista. Conectamos el disco USB al equipo.

#06. Apuntamos el identificador del disco que acabamos de conectar y pulsamos Ctrl-C para volver.

#07. Seleccionamos el disco USB de las opciones que nos da.

#08. Navegamos hasta el directorio donde se encuentran la imágenes y le damos a DONE.

#09. El programa montará la carpeta y hará una serie de comprobaciones. Cuando termine pulsamos ENTER.

#10. Escogemos ALL y pulsamos ENTER. Para la otra opción habria que saber la IP o MAC de los equipos a restaurar, y con las pruebas que ha dado tiempo a hacer en clase no ha funcionado.

#11. Seleccionamos BEGGINER y pulsamos ENTER.

#12 Escogemos RESTORE-DISK para restaurar el dico completo de los clientes que se conectarán.

#13 Para ahorrar algo de tiempo, seleccionamos NO y pulsamos ENTER para saltarnos la verificación de la imagen.

#14 Decidimos qué queremos que haga el equipo cuando termine de restaurarse. Hemos seleccionado REBOOT.

#15 Seleccionamos la imagen a restaurar.

#16 Seleccionamos el disco duro donde se escribirá la imagen.

#17 Elegimos el modo de difusión de los datos de la imagen por la red. Hemos elegido MULTICAST.

#18 Elegimos la forma de decidir cuando empezar la transmisión. Elegimos CLIENTS-TO-WAIT para que comienze cuando se conecte un determinado númoero de clientes.

#19 Escribimos el número de equipos que queremos restaurar y pulamos ENTER.

#20 Nos muestra una advertencia y preguna si queremos continuar. Pulsamos ENTER para confirmar.

Ahora continuamos en la siguiente sección ya en el portátil a restaurar.

Restaurar imagen

Ahora es el momento de restaurar el equipo conectándonos al servidor de imágenes, que ya está a la espera. Para ello iniciaremos el equipo desde la tarjeta de red, pero primero hemos tenido que realizar los pasos de las secciones anteriores.

#01. Arrancamos el ordenador a restaurar desde la tarjeta de red.

Pulsamos ENTER para continuar.

#02. Aparecerá el menú de arranque de Clonezilla Server (DRBL). No tocamos nada.

#03. Volvemos un momento al equipo servidor y pulsamos ENTER.

#04. En el menú de arranque de DRBL del equipo a restaurar aparecerá una nueva opción. La seleccionamos y confirmamos con ENTER.

#05. Ahora comenzará el proceso automático de restauración. Cuando termine se reinciará el equipo tal como se encontraba en el momento que se hizo la imagen.

#06. Por desgracia el proceso ha dado un error mientras se restauraba la imagen y no ha dado tiempo a averiguar cuál ha sido el problema.
